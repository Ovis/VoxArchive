# 実行ポリシー

## 目的
このプロジェクトは、会議録音アプリ（Windows/WPF）の長期開発を継続的に進める。

## 自律実行ルール
1. Codex は原則としてユーザー確認なしで、ローカル作業を継続する。
2. 判断が必要な場合は、仕様書・既存計画・過去ログに従って最小リスクの選択を行う。
3. 破壊的操作（履歴改変やファイル大量削除）は行わない。

## 安全制約
1. 作業はこのPCローカルのワークスペース内に限定する。
2. 外部システム変更、外部公開、リモート更新は行わない。
3. ネットワーク依存が必要な作業は後回しにし、ローカルで完結する代替を優先する。

## Git運用
1. 作業ブランチは `zissou` を使用する。
2. 変更は機能単位で小さくコミットする。
3. コミットタイトルは日本語で記述する。`種別(対象): 要約` 形式を推奨する。

## セッション運用
1. セッション開始時に `docs/project/session-log.md` と `docs/project/backlog.md` を確認する。
2. セッション終了時に、実施内容・結果・次アクションを `session-log.md` に記録する。
3. 重要な方針変更は `docs/project/roadmap.md` に反映する。
