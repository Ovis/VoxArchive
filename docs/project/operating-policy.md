# 開発運用ポリシー

## 1. ブランチ運用
- 機能集約ブランチ: zissou
- main 反映前に以下を満たす:
  - docs 同期
  - NUnit テスト通過
  - 手動 E2E チェック通過

## 2. コミット方針
- 日本語タイトル
- 1コミット1目的（UI修正/機能追加/設計変更を分離）

## 3. ローカル限定原則
- 作業はローカル環境に限定
- 外部サービスへの影響が出る操作は実施しない

## 4. 品質基準
- クラッシュ時に設定・カタログが破損しにくい構造を優先
- フォールバック（Process -> Speaker）を維持
- 例外はログ化し、UI を可能な限り継続
