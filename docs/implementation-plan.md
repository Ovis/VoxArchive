# 実装計画（2026-03-17 時点）

## 1. 方針
現在は MVP 一巡を完了しており、以降は以下を優先する。
- 安定化（録音停止/フォールバック/永続化の堅牢性）
- 保守性（共通スタイル、例外処理、ログの統一）
- 品質保証（NUnit テストの拡充）

## 2. 直近フェーズ
### Phase A: 安定化
- Process Loopback の失敗理由を分類し、フォールバック経路を維持
- 録音停止時の後処理（ライブラリ反映・自動文字起こし）の再試行系を維持
- JSON カタログ compact 条件の実測チューニング

### Phase B: 品質改善
- WPF 画面間のスタイル共通化を進める
- UI で未使用の状態値・重複ロジックを削減
- 例外ログ粒度の統一

### Phase C: テスト拡張
- 録音モード別の統合テスト追加
- ライブラリ操作（リネーム重複、欠損除外、モノラル保存）の自動テスト追加
- 文字起こしキューの同時投入・重複抑止テスト追加

## 3. 完了定義
- 主要機能の回帰テストが NUnit で常時通過
- 録音～ライブラリ～文字起こしまで E2E で手順化済み
- docs 配下が実装と一致
