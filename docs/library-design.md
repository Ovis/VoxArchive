# ライブラリ設計

最終更新: 2026-03-17

## 1. 目的
録音ファイルをアプリ内で管理し、再生・編集・削除・文字起こし操作を行う。

## 2. UI 構成
- 左: 録音ファイル一覧（DataGrid）
- 右: 再生/編集ペイン
- 下: ステータス表示

## 3. 一覧操作
- ファイル追加（任意 FLAC）
- 欠損ファイル一括削除
- チェック行の一覧一括削除
- チェック行のファイル一括削除
- 右クリックメニュー（Explorer/文字起こし/削除系）

## 4. 再生操作
- 再生/停止
- シークバー直接移動
- ステップ前後シーク
- 倍速変更 + 等倍復帰
- Speaker/Mic 個別ゲイン調整
- ミックスしてモノラル再生

## 5. 編集操作
- タイトル保存（FLAC タグ）
- ファイル名リネーム
  - 同名存在時は警告ダイアログ
- 一覧から削除（実体保持）
- ファイル削除（実体削除）

## 6. モノラル保存
- 現在ゲインでモノラル化して保存
- 形式: WAV / MP3 / FLAC
- ffmpeg を利用（MP3/FLAC 変換）

## 7. カタログ永続化
RecordingCatalogService が担当。

### 7.1 ファイル構成
- library.json（スナップショット）
- library.json.bak（バックアップ）
- library.ops.jsonl（操作ログ）

### 7.2 更新モデル
- 追加/更新/削除/リネームを JSONL 操作ログとして追記
- 閾値到達でスナップショット再生成（compact）

### 7.3 クラッシュ耐性
- append + flush to disk
- 原子的差し替え（temp + replace）
- snapshot 破損時 bak 読み込み
- ops の壊れ行スキップ

## 8. ライブラリ表示中のメモリキャッシュ
- AcquireInteractiveSession() で表示期間のみキャッシュを保持
- ウィンドウ破棄でセッション解放
- 非表示時はキャッシュを持たず、必要時ロード
